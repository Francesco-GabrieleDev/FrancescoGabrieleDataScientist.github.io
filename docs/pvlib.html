Problem / Goal

Automatically identify POI areas for mobility analysis so activity can be summed within well-defined boundaries.

Input / Output

Input: Table of POI IDs with longitude/latitude.
Output: Polygon geometries for each POI.

Method

  • Retrieve imagery centered on each POI via API.
  • Map pixels to geographic coordinates.
  • Segment POI areas with YOLOv11.
  • Convert segmented pixels into polygons.

Result / Use

Counted mobility pings within the polygons to measure activity for analysis or monitoring dashboards.

Skills / Tools

Python, YOLOv11, geo-mapping, API integration, spatial data processing.
