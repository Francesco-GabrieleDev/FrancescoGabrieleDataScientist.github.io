Fine-Grained Mobility Prediction for Retail Demand using Graph Neural Networks

Problem Addressed
Classical gravity-based mobility models break down at fine resolutions. This project predicts incoming human mobility at the parcel level — specifically for individual gas station sites.

Why It Matters
Human mobility pings within station canopies show a ~0.7 correlation with actual sales, making mobility a strong proxy for demand.

Model Performance

  • Mean absolute error of ~19 pings per station.
  • Graph-based models outperformed non-graph baselines by explicitly modeling spatial context and local competition.

(Metrics evaluated at station level across ~2.8k sites)

Business Impact

  • Established a framework for white-space analysis to support new site placement.
  • Enabled site-level demand estimation without direct access to sales labels.
  • Captured competitive spillovers and spatial dependencies missed by classical mobility models.

Unified Geospatial Mapping Platform

Problem Addressed
Built a unified mapping platform that lets data scientists move geospatial data and ML outputs from Python directly into an interactive map for exploration, validation, and sharing.

Why It Matters
Geospatial workflows typically split across notebooks, databases, and BI tools. This tool removes the handoff friction so teams can iterate faster, debug spatial issues earlier, and communicate results clearly.

Model Performance
Model metrics are tracked in Python; predictions can be uploaded and visualized immediately on the map for spatial QA and stakeholder review.

Business Impact

  • Reduced time from model/data output to map visualization from hours to minutes.
  • Enabled self-serve spatial analysis with filters, charts, and widgets without extra engineering.
  • Allowed direct SQL querying on map datasets to generate new layers for decision-making.

Platform Capabilities (Python + Map)

  • Python - Map bridge: upload a geodataframe and create a map in one call.
  • Database integration: store datasets safely and reuse them across maps.
  • Model upload: push a scikit-learn pipeline and run predictions against map data.
  • Interactive analytics: filters, charts, and widgets for exploration.
  • SQL on the map: run queries directly in the UI and add results as new layers.

Milan Morphometric Urban Typology

Problem Addressed
Developed an automated pipeline to classify every building in Milan into distinct urban morphology types using open geospatial data and unsupervised clustering.

Why It Matters
The resulting typology map encodes the physical character of each neighbourhood, enabling spatial feature enrichment for any downstream model that needs to understand where something is beyond raw coordinates.

Pipeline Scale
63,210 buildings processed · 12 morphometric characters · k = 1–11 evaluated · Full city of Milan covered end-to-end

Business Impact

  • Tell apart a dense historic block from a suburban sprawl without ever opening a map — just query the cluster label.
  • Find every area in Milan that looks like Isola or Brera in seconds, at city scale.
  • Drop the cluster and morphometric features into any location-based model to instantly improve where-awareness — without collecting new data.

POI Segmentation from Satellite Imagery (YOLOv11)

Problem Addressed
Developed an automated framework to extract polygon geometries of points of interest from satellite imagery using a YOLOv11 segmentation model.

Why It Matters
The extracted geometries enabled accurate measurement of human activity within each POI area.

Model Performance
mAP@50–90: 0.57 · mAP@50: 0.96

Business Impact

  • Eliminated the need for third-party manual annotation, reducing operational costs.
  • Enabled large-scale geometry extraction (10k+ POIs processed in ~1 hour).
  • Learned compact POI representations reusable for downstream modeling tasks.

AI-Powered Wind Farm Design Assistant

The Challenge
Designing a wind farm requires data retrieval, simulations, and engineering calculations, which typically demand coding skills and significant domain knowledge. This creates a barrier for users who want guidance and actionable results without building the entire computational pipeline themselves.

How Reex Energy Agent Assistant Solves It
The Reex Energy Agent Assistant combines LLM-based reasoning with engineering Python libraries and live data access. It autonomously runs simulations, performs calculations, and guides decision-making while you interact through chat. Unlike a standard LLM that only suggests steps or generates code, it executes the full workflow end-to-end and delivers concrete, data-driven design recommendations.

Map-Based Renewable Energy Simulation Platform

Business Impact

  • Reduced time to evaluate candidate sites by combining spatial selection, data extraction, and energy modeling in one place.
  • Enabled rapid scenario testing (turbine choice, hub height, wake losses, Weibull factors).
  • Provided stakeholder-ready outputs with immediate spatial context and on-map results.

Platform Capabilities (Reex Extension)

  • Map-native site selection: draw an area on the map and intersect with energy datasets stored in the database.
  • Wind turbine catalog integration: select from a built-in turbine library (power curves, Cp curves, rotor diameter).
  • Single turbine & farm modes: compute AEP and capacity for individual turbines or multi-turbine farms.
  • Interactive controls: adjust hub height, terrain roughness, wake losses, and distribution parameters.
  • Results on the map: compute and visualize outputs without leaving the Reex environment.

Use Cases

  • Wind farm site selection based on mean wind speed and terrain constraints.
  • Pre-feasibility screening for renewable projects with rapid scenario testing.
  • Stakeholder communication using map-based visuals and outputs.
